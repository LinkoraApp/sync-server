###SYNC_SERVER_INFO###

This server is meant to communicate only with the Linkora App (Android & Desktop).There's no web frontend yet, so you can’t control anything via browser UI.

If you're technical and want to build a custom solution that connects to this server, that’s totally possible. Consider mailing linkoraapp@protonmail.com so I can link your frontend or anything useful into the ecosystem via the README.

Database & Data Syncing

  • Apps have their own local databases.
  • This server helps store the data and provides syncing capabilities.

Deletion Behavior

  • If something is deleted on the app, and the server is added to the app, it will be deleted on sync — even if the server was offline when the delete happened.
  • It cannot be undone.
  • (A trash system might come later, but for now, deletion is permanent.)

LWW (Last-Write-Wins) Implementation

  • Server uses LWW logic.
  • Clients keep their own updates while the server is offline.
  • Once reconnected, the latest write wins and syncs to server DB.

Blank Pages on Other Routes

  • Blank pages are expected on all routes except this one.
  • All other routes require a valid auth token.
  • Without it, no access is granted.
